  </Tab>
  <Tab title="yarn">
    ```bash
    yarn add kundera
    ```
  </Tab>
</Tabs>

## Module Formats

Kundera ships as ESM with TypeScript declarations. CommonJS is supported via interop.

```typescript
// ESM (recommended)
import { createHttpProvider } from 'kundera/provider';

// CommonJS
const { createHttpProvider } = require('kundera/provider');
```

## Subpath Exports

For optimal tree-shaking, import from specific subpaths:

```typescript
import { createHttpProvider } from 'kundera/provider';
import { Felt252, ContractAddress } from 'kundera/primitives';
import { encodeCalldata, decodeEvent } from 'kundera/abi';
import { createAccount } from 'kundera/account';
import { getContract } from 'kundera/contract';
import { loadWasmCrypto } from 'kundera/crypto';
```

<Tip>
Subpath imports ensure bundlers only include what you use. The main `kundera` export re-exports everything, which may increase bundle size.
</Tip>

## Verify Installation

```typescript
import { createHttpProvider } from 'kundera/provider';

const provider = createHttpProvider({
  url: 'https://starknet-mainnet.public.blastapi.io',
});

const { result } = await provider.starknet_chainId();
console.log('Connected to chain:', result);
// Output: 0x534e5f4d41494e (SN_MAIN)
```

## Runtime Requirements

| Runtime | Version | Notes |
|---------|---------|-------|
| **Bun** | 1.0+ | Native crypto via FFI (fastest) |
| **Node.js** | 18+ | Requires `ffi-napi` for native, else WASM |
| **Browsers** | Modern | WASM crypto only |
| **Deno** | 1.30+ | WASM crypto only |

<Warning>
## Crypto Backend

Kundera requires a crypto backend for signing and hashing. See [Native Crypto](/getting-started/native) or [WASM Crypto](/getting-started/wasm) for setup.
</Warning>
