Skills are copy-paste patterns for common Starknet tasks. Each skill is a self-contained recipe that composes Kundera's primitives into a working solution. Start with a transport, add a provider, then build up to contract reads, writes, and account operations.

**Composition Flow:** Transport → Provider → Contract Read/Write → Account Operations

## Why Skills?

| Benefit | Description |
|---------|-------------|
| **No dependencies** | Skills use only Kundera core APIs |
| **Copy-paste** | Take what you need, modify as required |
| **Well-tested** | Each skill includes a test suite |
| **Tree-shakeable** | Import only what you use |

<Tip>
For **type-safe** contract interaction with auto-completed function names and inferred argument types, start with the [Get Contract](/typescript/skills/get-contract) skill. It uses ABI type inference (`InferArgs`, `InferReturn`, `InferFunctionName`) to catch errors at compile time.
</Tip>

## Using Skills

### 1. Copy the Code

```bash
# Copy a skill to your project
cp -r examples/skills/http-provider src/skills/
```

### 2. Install Peer Dependencies (if any)

Some skills have optional peer dependencies:

```bash
# Wallet Modal requires one of:
npm install @starknet-io/starknet-kit
# or
npm install get-starknet
```

### 3. Import and Use

```ts
import { createHttpProvider } from './skills/http-provider';

const provider = createHttpProvider({ url: 'https://api.zan.top/public/starknet-sepolia' });
```

## Skill Structure

Each skill follows this structure:

```
examples/skills/<skill-name>/
├── index.ts              # Main implementation
├── README.md             # Usage documentation
└── <skill-name>.test.ts  # Test suite
```

## Adapt as Needed

Skills are starting points, not rigid frameworks. Feel free to modify the code to fit your project's patterns and requirements.
